# Exploring the GitHub web UI — profile, repo creation, and issues

I clicked around github.com to figure out the profile page, making a repo, and the issue tracker.

## Profile page

My profile shows pinned repos, a contribution grid (color-coded by commits), and my bio. I pinned six repos to show what I'm working on. The activity log on the right lists recent stars and forks — handy for remembering what I looked at last week.

## Repository creation

The green "New" button opens a form: name, description, public/private, README checkbox, .gitignore and license dropdowns. I checked "Add a README" to skip the initial CLI push. The `.gitignore` has templates by language. Repo names only need to be unique under my username, not globally — I tried "test" and it errored until I realized that.

## Issue tracker

The Issues tab shows "No issues open" with a "New issue" button. I created one — title, body, then it appeared as `#1` with an "Open" badge. Filters at the top let me sort by date, comments, or reactions. Issue numbers are per-repo, so `#1` in one repo is different from `#1` in another.

## What I'd try next

Link a project board to issues and try label management.
