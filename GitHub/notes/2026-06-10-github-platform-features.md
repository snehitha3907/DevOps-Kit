# Exploring GitHub platform features — wiki, projects, and insights

I went beyond the basic repo view today to check out the other tabs GitHub provides.

**Wiki** — each repo can have its own wiki. I clicked the Wiki tab on a few popular repos. Some use it for full documentation, others just have a landing page. You edit it just like a repo — it's a separate Git repo actually. I could see how this would be useful for project docs that aren't in the code itself.

**Projects** — this is a kanban-ish board for tracking work. I created a test project board in one of my repos, added a few columns (To Do, In Progress, Done), and dragged cards around. It links to issues and PRs automatically if you paste the URL. Reminds me of Trello but built into GitHub.

**Insights** — the Insights tab shows repo traffic, contributions, and dependency graphs. I checked the pulse page to see recent activity and the contributors graph to see who's been committing. The code frequency graph shows additions/deletions over time, which is cool for spotting when a big refactor happened.

Nothing tripped me up this time — the UI is pretty intuitive. I'll try linking project boards to actual issues next.
